## What was done

### Router expansion (router.py)
- 10 intents → 16 unique intents, ~35 regex patterns
- Added shared date token `_D` covering: today, yesterday, now, N days ago,
  weekday names (full + abbreviated), ISO dates
- Added shared range token `_R` covering: last/past N days, last/past week,
  last/past month, this week, this month
- New intents: `weight_range`, `nutrition_range`, `training_range`,
  `recovery_range`, `cardio_for`, `cardio_latest`
- New phrasings: "how am I doing", "give me a summary", "how did I sleep",
  bare "nutrition"/"calories"/"protein" (→ today), bare "sleep" (→ yesterday),
  "what's the plan today", "what did I lift monday", etc.

### LLM fallback classifier (handlers/classify.py)
- Router miss → Claude Haiku 4.5 with strict JSON contract
- Model picks from known intent set or returns "unknown"
- Hallucinated intent names are rejected
- Tested: "did I train legs recently" → training_latest,
  "what's my body fat percentage" → body_scan_latest,
  "when did I last do incline dumbbell press" → last_exercise w/ correct exercise

### Bug #4 fix: exercise fuzzy-match (handlers/query.py)
- `last_session_of_exercise` now tries exact match first, falls back to LIKE %term%
- "last bench" now correctly matches "Bench Press"
- Returns `matched_exercise` field so caller knows what actually matched

### Bug #5 fix: null-aware nutrition averages (handlers/query.py)
- `nutrition_range_summary` now skips NULL values in averages
- Reports `n_with_calories` and `n_with_protein` alongside `n` total rows
- A week with 3 missed days no longer reports 30% lower averages

### Bug #8 fix: stats_snapshot fallback (handlers/query.py)
- `stats_snapshot` tries today first for nutrition/recovery
- Falls back to most recent row if today has no data
- Reports `as_of: "today"` or `as_of: "most_recent"` so caller knows
- Also restructured return format: `nutrition: {data: ..., as_of: ...}`

### Test suite (tests/)
- conftest.py: in-memory SQLite fixture seeded with known data
- test_router.py: 73 tests covering all 16 intents + edge cases
- test_dates.py: 23 tests for date/range resolution with mocked today
- test_query.py: 28 tests including null-aware averages, fuzzy match, stats fallback
- All 124 tests passing

### Adversarial probing
